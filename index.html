解决css样式中“高度塌陷“”的方法

什么是高度塌陷？
在文档流中，父元素的高度默认是被子元素撑开的，也就是子元素多高，父元素就多高。但是当子元素设置浮动之后，子元素会完全脱离文档流，此时将会导致子元素无法撑起父元素的高度，导致父元素的高度塌陷。
<!DOCTYPE html>




高度塌陷的问题








1.可以将父元素的高度写死，以避免塌陷的问题的出现（但是，一旦高度写死，父元素的高度将不能自动适用于子元素的高度，所以这种方案不建议使用）
修改box1样式
.box1{
height: 200px;
width:200px;
background-color: yellow;
border: 10px red solid;
}
2.将父元素也设置浮动(使用这种方式开启，虽然可以撑开父元素，但是会导致父元素的宽度丢失而且使用这种方式也会导致下边的元素向上移)
修改box1样式
.box1{
background-color: yellow;
border: 10px red solid;
float: left;
}
3.设置元素为inline-block(可以解决塌陷问题，但会导致宽度丢失，不推荐使用这种方式);
修改box1样式
.box1{
background-color: yellow;
border: 10px red solid;
display: inline-block;
}
4.将元素的overflow设置为一个非visible的值(副作用最小的方式)
修改box1样式
.box1{
background-color: yellow;
border: 10px red solid;
overflow: auto;/或者hidden/
zoom:1;/兼容 IE6/
}

  1. 可以直接在高度塌陷的父元素的最后，添加一个空白的div(由于这个div并没有浮动，所以它是可以撑开父元素的高度的然后再对其进行清除浮动，这样可以同过这个空白的div来撑开父元素的高度基本没有副作用，缺点：使用这种方式虽然可以解决问题，但是会在页面中添加多余的结构。)
    将class为box3的div改为box1的子元素然后修改.box3样式（只针对本解决方式）
    .box3{
    clear:both;
}
  2. 通过after伪类向高度塌陷的父元素的最后添加一个空白的块元素，然后对其进行清除浮动(最推荐使用的方式，几乎没有副作用)
    修改box1样式
    .box1:after{
    content:””;
    display:block;
    clear:both;
    }
    .box1{
    zoom:1
    }
